# UX Profile: uconsole_desktop

## Screen Class

Desktop-class Linux handheld display.

## Input Model

Keyboard, pointer, GTK gestures, and desktop window controls.

## Feature Set

Full desktop handheld workflow with chat, map, dashboard, GPS, Team, settings,
diagnostics, and host integration.

## Screen Set

Dashboard, Chat, Map, Team, GPS, Tracker, Settings, Diagnostics.

## Map Mode

Full Linux map mode with filesystem/cache adapters, offline tiles, overlays, and
future route/tracker integration.

## Chat Mode

Full chat with richer desktop presentation potential.

## Team Action Mode

Full Team action surface with desktop-friendly controls.

## GPS Mode

Full GPS status and diagnostics with runtime scheduling owned outside renderer.

## Modal/Picker Strategy

GTK-friendly dialogs or desktop panels; LVGL fallback remains compatibility
only.

## Renderer Family

GTK primary, optional LVGL fallback for legacy surface.

## Deferred Decisions

GTK UX pack object model, rich map overlay controls, and package integration
remain deferred.

Board describes. Target chooses. UX Pack presents. Renderer draws.
